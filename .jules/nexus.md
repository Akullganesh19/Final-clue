## 2024-03-05 — Cross-Case Entity Dossier
**Product understood as:** A system to link cold cases and triage evidence.
**Derivation reasoning:** We store cases which contain entities (people, vehicles, locations, weapons). Users use this system to find linkages between cases. Therefore, users obviously need a way to see which entities appear across multiple cases chronologically — because tracking an entity's movement or recurrence across different incidents is a fundamental investigative technique. It doesn't exist because the focus has been on case-to-case similarity scores rather than entity-centric timelines. I'm building it because giving investigators an immediate timeline of a specific suspect, vehicle, or weapon across the entire database provides a completely new lens for generating leads.
**Feature built:** An entity tracker utility (`generateEntityDossiers`) and UI component (`EntityDossierView`) that automatically extracts overlapping entities across the case database, calculates their timespans, and plots their appearances in a chronological timeline.
**User impact:** Users can now immediately see cross-case entities and track their timeline of involvement across different incidents without having to manually cross-reference case files.
**Next logical feature:** Generating interactive geographic maps from the cross-case entity dossiers to track the physical movement of suspects or vehicles over time.
