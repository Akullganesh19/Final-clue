## 2025-07-28 тАФ Native Web Crypto API for Audit Ledger
**Risk identified:** The legacy audit ledger (`generateAuditHash`) used a custom 32-bit DJB2 bitwise hash combined with standard pipe delimiter injection `|` and `Math.random()` for IDs. This is highly susceptible to hash collisions (via the Birthday Paradox), is not cryptographically sound, and allows trivial chain-of-custody forgery in an evidence system.
**Migration target:** The ecosystem standard for secure client/server hashing and identifiers relies on the native Web Crypto API (`crypto.subtle.digest` with SHA-256) and `crypto.randomUUID()`. Data should be passed as arrays via `JSON.stringify` serialization to mitigate delimiter injection.
**Migrated this session:** Added asynchronous `generateSecureAuditHash` and `createSecureAuditLog` implementations using the Web Crypto API alongside the legacy synchronous functions. Marked legacy versions as `@deprecated`. Covered both with new testing infrastructure.
**Remaining:** Identifying call sites across the application that rely on the synchronous `createAuditLog` and migrating them to `await createSecureAuditLog(...)`. This will require updating the calling code chains (e.g. `EventBus` hooks or Agent outputs) to support asynchronous updates.
**Next session:** Find instances of `createAuditLog` outside `src/utils/audit.ts` using `grep -rn "createAuditLog" src/` and migrate the first major domain (e.g., the Audit Agent pipeline) to the async pattern.
